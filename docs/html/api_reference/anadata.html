1. Anadata module¶

This module can handle the analyzed data stored in Kaiseki server. (retrieving, registering, etc.) Retrieving method is implemeted in C++ modules (libana2), and retrieved dataset uses a xarray.Dataset.

class nifs.anadata.AnaData(shot_number, subshot=1)[source]¶

class to handle analyzed data stored in analysis (kaiseki) server. When retrieving or registering data to kaiseki server, wrapper class _AnaData which is implemeted in libana2 is used. Retrieved data is reconstructed as one dataset using xarray package

Todo

the method of registering a xarray.Datset to kaiseki-server has not been implemented yet.

Parameters

  • shot_number (int, required) – shot number

  • subshot (int, optional) – sub-shot number, by default 1

Example

from nifs.anadata import AnaData
# set (shot number: 80000, subshot number: 1) database
ana = AnaData(80000, 1)

retrieve(diagnostics: str) → xarray.core.dataset.Dataset[source]¶

Retrieving analyzed data from kaiseki-server. This method returns a xarray.Dataset object. If you want to know about xarray, please see the xarray documentation: http://xarray.pydata.org/.

Parameters

diagnostics (str) – name of diagnostics which is registered in the kaiseki-server.

Returns

dataset of a xarray object.

xarray.Dataset

from nifs.anadata import AnaData

# Retrieving dataset of thomson
ds = ana.retrieve("thomson")
